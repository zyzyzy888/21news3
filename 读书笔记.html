二、两个基本原则

原则一：编写明确和具体的指令

清晰≠短小

可以使用任何明显的标点符号将特定的文本部分与提示的其余部分分开。这可以是任何可以使模型明确知道这是一个单独部分的标记。使用分隔符是一种可以避免 提示注入 的有用技术。

提示注入是指如果用户将某些输入添加到提示中，则可能会向模型提供与您想要执行的操作相冲突的指令，从而使其遵循冲突的指令而不是执行您想要的操作。即，输入里面可能包含其他指令，会覆盖掉你的指令。对此，使用分隔符是一个不错的策略。

策略1:使用分隔符
  • 三重引号:"""
  • 三重反题:```
  • 三破折号:---
  • 尖括号:<>
  • XML标记:
策略2:要求结构化输出HTML，JSON

第二个策略是要求生成一个结构化的输出，这可以使模型的输出更容易被我们解析，例如，你可以在 Python 中将其读入字典或列表中。

策略3:检查条件是否满足检查完成任务所需的假设

如果任务做出的假设不一定满足，我们可以告诉模型先检查这些假设，如果不满足，指示并停止执行。还可以考虑潜在的边缘情况以及模型应该如何处理它们，以避免意外的错误或结果。

策略4:提供少量示例

即在要求模型执行实际任务之前，提供给它少量成功执行任务的示例。

原则二 ：给模型足够的时间来思考

策略1:指定完成一项任务的步骤
第一步:...
第二步:…
第N步:...
策略2:在匆忙得出结论之前，指导模型找出自己的解决方案

第二个策略是要求生成一个结构化的输出，这可以使模型的输出更容易被我们解析，例如，你可以在 Python 中将其读入字典或列表中。

三、局限性

虚假知识

模型偶尔会生成一些看似真实实则编造的知识。
如果模型在训练过程中接触了大量的知识，它并没有完全记住所见的信息，因此它并不很清楚自己知识的边界。这意味着它可能会尝试回答有关晦涩主题的问题，并编造听起来合理但实际上并不正确的答案。我们称这些编造的想法为幻觉。

例如: 模型可能会从真正的牙刷公司中创造一个虚构的产品名称描述。

减少幻觉

为了减少模型生成的幻觉内容，可以采取一些策略。

一种有效的方法是要求模型首先从文本中找到任何相关的引用，然后要求它使用这些引用来回答问题，并确保答案可以追溯到源文档。

第三章 迭代

一、快速指南

在这一部分，我们将讨论迭代过程的核心要素，包括明确和具体的提示编写，对结果进行分析，提炼想法和提示，以及不断重复这一过程。

关键步骤

1. 尝试编写初步提示。

2. 分析为什么结果没有给出预期的输出。

3. 根据分析提炼想法和编写更精细的提示。

4. 重复上述步骤，不断优化提示直至得到满意的结果。

第四章 摘要

一、引言

深入了解代码，以了解如何使用它来总结文本是提升处理大量信息能力的关键步骤。文本摘要作为大模型的重要应用场景，对于快速获取文本核心信息至关重要。

为了进行文本摘要，我们首先需要引入OpenAI包，加载API密钥，并定义相应的getCompletion函数。

二、单一文本概括 Prompt 实验

在电子商务网站中，处理大量用户评论是一个常见的需求。使用大模型进行文本摘要能够有效地概括这些评论，帮助商家和平台快速了解客户反馈。

1. 限制输出文本长度

在生成摘要时，限制输出文本的长度是非常重要的。例如，我们可以尝试将摘要长度限制为最多30个词。

2. 关键角度侧重

根据业务需求，我们可以调整Prompt来侧重不同的关键角度。例如，商家可能更关心商品质量和价格，而平台可能更关注整体服务体验。

3. 关键信息提取

除了侧重特定角度，有时我们还需要从文本中提取特定信息。通过调整Prompt，我们可以要求LLM仅提取特定角度的信息，并过滤掉其他内容。

三、多条文本概括 Prompt 实验

在实际应用中，我们通常需要处理多条评论文本。通过循环调用“文本概括”工具，我们可以依次处理每一条评论。然而，对于大量评论，使用循环可能不是最高效的方法，因此需要考虑使用整合评论和分布式计算等方法来提升运算效率。

第五章 推理

在本章中，我们将探讨如何使用大型语言模型从产品评论和新闻文章中推断情感和主题。这些任务不仅有助于我们理解文本内容，还能为应用程序的开发提供便利。

一、情感（正向/负向）

通过编写特定的prompt，我们可以让大型语言模型识别文本中的情感倾向，并以简洁的方式输出结果。例如，通过添加指令，模型可以返回一个表示正面或负面情感的单词，从而方便后续处理。

二、识别感情类型

大型语言模型能够从文本中提取特定信息，这对于理解客户对产品的看法非常有帮助。通过分析评论中的情感表达，我们可以获取更多关于客户满意度的信息。

三、识别愤怒

对于企业而言，了解客户是否表达了愤怒情绪至关重要。通过分类问题的方式，我们可以判断评论中是否包含愤怒情绪，从而及时采取措施解决客户问题。

四、从客户评论中提取产品和公司名称

信息提取是自然语言处理的重要部分，它涉及从文本中提取关键信息。在电子商务网站中，提取产品和公司名称有助于我们了解客户的购买偏好和市场趋势。

五、一次完成多项任务

通过编写单个prompt，我们可以同时提取多种信息，如情感、主题等。这种方法不仅提高了效率，还使得结果更加全面和准确。

六、推断主题

对于新闻文章等文本内容，推断主题是一个重要的任务。通过编写特定的prompt，我们可以确定文本中涵盖的主题，并为其打上相应的标签。这对于新闻分类、内容推荐等应用具有重要意义。

第六章 转换

一、引言

大型语言模型（LLM）具有强大的文本转换能力，能够轻松实现多语种文本翻译、拼写及语法纠正、语气调整、格式转换等功能。在本章中，我们将介绍如何使用编程方式调用API接口来实现这些功能。

二、文本翻译

LLM可以轻松地将文本从一种语言翻译成另一种语言。以下是几种常见的文本翻译场景：

1. 中文转西班牙语

使用LLM，我们可以轻松地将中文文本翻译成西班牙语。

2. 识别语种

LLM可以识别给定文本的语种，为后续的翻译工作提供便利。

3. 多语种翻译

LLM支持多种语言的互译，使得跨国交流更加便捷。

4. 翻译+正式语气

在翻译的同时，LLM还可以调整文本的语气，使其更符合目标语种的正式场合表达习惯。

5. 通用翻译器

随着全球化的发展，一个能够识别并翻译多种语言的通用翻译器变得越来越重要。它可以帮助我们实现更方便的跨国交流。

三、语气/风格调整

LLM可以根据不同的受众对象调整写作的语气和风格。例如，对于工作邮件，我们可以使用正式语气和书面用词；而对于微信聊天，则可以使用更轻松、口语化的语气。

四、格式转换

ChatGPT等LLM擅长不同格式之间的转换。例如，我们可以将JSON格式的数据转换为HTML、XML或Markdown等格式，以适应不同的应用场景。

五、拼写及语法纠正

LLM还可以帮助我们检查和纠正拼写及语法错误。这对于使用非母语语言的人来说尤其重要，特别是在撰写论文或其他正式文档时。

第七章 扩展

文本扩展是将短文本，例如一组说明或主题列表，输入到大型语言模型中，让模型生成更长的文本，例如基于某个主题的电子邮件或论文。这样做有一些很好的用途，例如将大型语言模型用作头脑风暴的伙伴。但这种做法也存在一些问题，例如某人可能会使用它来生成大量垃圾邮件。因此，当你使用大型语言模型的这些功能时，请仅以负责任的方式和有益于人们的方式使用它们。

在本章中，将学会如何基于 OpenAI API 生成适用于每个客户评价的客户服务电子邮件。我们还将使用模型的另一个输入参数称为温度，这种参数允许您在模型响应中变化探索的程度和多样性。

一、环境配置

为了使用OpenAI API进行文本扩展，我们需要配置相应的环境。这通常涉及到安装必要的库、设置API密钥等步骤。

二、定制客户邮件

我们将根据客户评价和情感撰写自定义电子邮件响应。因此，我们将给定客户评价和情感，并生成自定义响应即使用 LLM 根据客户评价和评论情感生成定制电子邮件。

三、使用温度系数

接下来，我们将使用语言模型的一个称为“温度”的参数，它将允许我们改变模型响应的多样性。可以将温度视为模型探索或随机性的程度。

例如，在一个特定的短语中，“我的最爱食品”最有可能的下一个词是“比萨”，其次最有可能的是“寿司”和“塔可”。因此，在温度为零时，模型将总是选择最有可能的下一个词，而在较高的温度下，它还将选择其中一个不太可能的词，在更高的温度下，它甚至可能选择塔可，而这种可能性仅为五分之一。您可以想象，随着模型继续生成更多单词的最终响应，“我的最爱食品是比萨”将会与第一个响应“我的最爱食品是塔可”产生差异。因此，随着模型的继续，这两个响应将变得越来越不同。 一般来说，在构建需要可预测响应的应用程序时，我建议使用温度为零。在所有课程中，我们一直设置温度为零，如果正在尝试构建一个可靠和可预测的系统，应该选择这个温度。如果尝试以更具创意的方式使用模型，可能需要更广泛地输出不同的结果，那么可能需要使用更高的温度。
